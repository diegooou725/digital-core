¿Qué es exactamente una migración Data Cloud?

Muchos creen que trabajar en la nube es simplemente arrastrar un archivo de Excel a Google Drive, pero al hacerlo, muchas fórmulas se rompen, los formatos se desconfiguran y la seguridad se pierde.

Nuestro trabajo consiste en tomar tus herramientas locales y reconstruirlas nativamente para Google Sheets o Excel Online (Microsoft 365). Además, configuramos arquitecturas de seguridad avanzadas: por ejemplo, logramos que el vendedor solo pueda editar la columna de "Ventas", mientras que el contador es el único que puede ver y editar la columna de "Ganancias".

Alcances ¿Qué INCLUYE el servicio?

  • Traducción de Fórmulas: Adaptamos las funciones de tu Excel local a sus equivalentes en la nube para garantizar que nada deje de calcularse.
  • Control de Permisos Avanzado: Asignamos niveles de acceso por correo electrónico. Tú decides quién puede ver, quién puede editar y quién tiene bloqueadas ciertas hojas o celdas.
  • Estructuración Colaborativa: Rediseñamos el flujo de captura para que varios usuarios trabajen simultáneamente sin estorbarse.
  • Historial de Versiones: Activación y capacitación para que puedas rastrear exactamente quién borró o modificó un dato y a qué hora lo hizo.

Lo que NO incluye este servicio

  • Pago de licencias mensuales corporativas (Google Workspace o Microsoft 365). El cliente debe contar con sus propias cuentas gratuitas o de paga.
  • Sistemas para manejar cientos de miles de registros diarios. (Para volumen industrial se requiere Data Hub con bases de datos reales).
  • Programación de código en la nube (Google Apps Script). (Corresponde a Data Macro).

Términos y Condiciones de Migración Cloud

1. Cuentas e Infraestructura: El cliente es el único titular y responsable de las cuentas de Google Workspace o Microsoft 365 utilizadas para alojar la información. Digital Core no asume responsabilidad por la pérdida de acceso, olvido de contraseñas o falta de pago de las licencias por parte del cliente a los proveedores (Google/Microsoft).

2. Limitaciones de Plataforma: La migración de herramientas locales (Excel Desktop) a entornos web (Google Sheets / Excel Online) está sujeta a las limitaciones técnicas propias de la nube. Algunas funcionalidades locales extremadamente complejas o dependientes de hardware específico podrían no tener un equivalente exacto en la versión web.

3. Responsabilidad Administrativa: Tras la entrega y configuración de los permisos de acceso y edición, es responsabilidad del administrador designado por la empresa cliente gestionar las altas o bajas de usuarios futuros en la plataforma. Solicitudes posteriores de soporte para gestión de usuarios se cotizarán como mantenimiento.

¿Qué es una arquitectura Front-End / Back-End?

Para lograr que varias computadoras trabajen juntas sin errores, no podemos usar un archivo normal. Utilizamos una técnica de ingeniería de software llamada "División de Base de Datos".

El Back-End (El cerebro): Es un archivo que solo contiene las tablas (la información pura) y se guarda en la computadora principal de tu oficina o en un disco de red compartido.

El Front-End (La cara visual): Es la interfaz con los botones y pantallas. Esta "cara" se instala en la computadora de cada empleado y se conecta a través de tu red local (Wi-Fi o cable) hacia el cerebro central. Así, cada empleado tiene su propia pantalla independiente, pero todos alimentan el mismo cerebro.

Alcances ¿Qué INCLUYE el desarrollo?

  • División Estructural: Programación y separación de la interfaz gráfica y las tablas de almacenamiento de datos.
  • Trabajo Simultáneo: Capacidad para que múltiples usuarios en la misma oficina agreguen, editen o busquen información en tiempo real.
  • Control de Concurrencia: Si dos empleados intentan editar el mismo registro (ej. el teléfono del mismo cliente) exactamente al mismo segundo, el sistema bloquea temporalmente a uno para evitar que la base de datos se corrompa.
  • Vistas Personalizadas: Configuración inicial para que la computadora de ventas solo vea el formulario de ventas, y la computadora gerencial vea los reportes completos.

Lo que NO incluye este servicio

  • Soporte o instalación de hardware (No vendemos ni configuramos el router, los cables de red física, ni las computadoras de tu oficina).
  • Conexión por internet para sucursales lejanas. (Este sistema requiere que las PCs estén en la misma red Wi-Fi o local. Para conectar sucursales remotas se requiere Hub SQL).
  • Lógica de almacén avanzada con lectores de códigos de barras. (Corresponde a Hub Stock).

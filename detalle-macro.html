¿Qué es exactamente una Macro?

Una macro es un fragmento de código (escrito en VBA para Excel o Apps Script para Google Sheets) que funciona como un pequeño robot integrado en tu archivo.

A diferencia de las fórmulas tradicionales (que solo calculan números), las macros pueden tomar acciones: pueden abrir otros archivos, copiar información de una hoja a otra, generar documentos PDF, enviar correos electrónicos de forma automática, o limpiar una base de datos nueva con solo presionar un botón.

Alcances ¿Qué INCLUYE el desarrollo?

  • Procesos de 1 Clic: Botones interactivos dentro de tu hoja de cálculo que disparan secuencias complejas de acciones.
  • Cruces de Información Masiva: Consolidación de datos de 5, 10 o 50 archivos diferentes en un solo reporte maestro en segundos.
  • Generación de Entregables (PDF): Programación para que el sistema tome los datos de una cotización o recibo, genere un PDF con tu logo y lo guarde en una carpeta específica.
  • Formatos Automáticos: Limpieza de datos (como lo que hacemos en Data Cleanse) pero programada para que ocurra automáticamente cada vez que pegues información nueva.

Lo que NO incluye este servicio

  • Desarrollo de software independiente (.exe) fuera del entorno de Microsoft Office o Google Workspace.
  • Extracción de datos de páginas web externas (Web Scraping). (Corresponde al paquete Data Connect).
  • Sistemas de bases de datos relacionales con múltiples niveles de seguridad de usuario. (Corresponde a Data Hub).

Términos y Condiciones de Automatización (Macros)

1. Estructura Inviolable: La automatización se diseña basándose en una estructura de columnas y hojas predefinida. Si el cliente altera dicha estructura original (agregando, eliminando, moviendo o renombrando columnas/hojas), la macro presentará errores fatales. La reparación de código por alteraciones del usuario no está cubierta por la garantía y generará costos de mantenimiento.

2. Entorno y Actualizaciones: El código (VBA o Google Apps Script) se programa para la versión actual del software. Digital Core no se hace responsable si futuras actualizaciones de seguridad de Microsoft Windows, Office 365, o Google Workspace bloquean o alteran la compatibilidad de las macros previamente entregadas.

3. Permisos de Seguridad: Es responsabilidad del usuario final habilitar la ejecución de Macros o conceder los permisos de aplicaciones de terceros (en el caso de Google) que el sistema operativo solicite para permitir el funcionamiento del código.
